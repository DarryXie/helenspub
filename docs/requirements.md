# 鸡尾酒数据库项目功能需求文档

## 1. 项目概述

### 1.1 项目名称
鸡尾酒数据库（Cocktail Database）

### 1.2 项目目标
建立一个基于 MySQL 的鸡尾酒内容管理系统，用于统一记录和展示鸡尾酒的基础信息、描述、配方、图片等内容，并支持服务人员在业务前台登录后创建待制作任务、查看配方和更新制作状态。

系统分为三部分：

- 前台公开端：主要用于展示鸡尾酒菜单和详情内容，用户可直接通过 `/` 或 `/menu` 访问，需优先适配手机屏幕。
- 前台业务端：供管理员和服务员登录后使用，统一从 `/app/login` 进入，主要处理待制作任务和查看配方。
- 后台管理端：Web 网页管理后台，主要用于维护鸡尾酒、分类、标签、原料、图片等主数据。

### 1.3 一期范围
一期以“菜单展示 + 内容管理 + 服务人员待制作管理”为核心，不包含在线下单、支付、会员积分等复杂业务。

### 1.4 终端要求

- 前台公开端和前台业务端需适配手机屏幕，优先考虑移动端使用体验。
- 后台管理端为桌面 Web 页面，适合电脑端操作。

## 2. 建设目标

- 建立结构化的鸡尾酒数据库。
- 支持后台高效录入、编辑、删除和管理鸡尾酒内容。
- 支持前台按菜单形式浏览鸡尾酒。
- 支持前台业务人员登录后创建和维护待制作任务。
- 支持图片上传与展示。
- 为后续扩展客户用户、收藏、下单、多语言等功能预留空间。

## 3. 用户角色

### 3.1 系统管理员

- 登录后台管理端。
- 管理鸡尾酒、分类、标签、原料、图片等内容。
- 管理前台业务用户账号。
- 在前台业务端也可查看配方和处理待制作任务。

### 3.2 服务员

- 通过前台业务端登录。
- 发起点单并创建待制作任务。
- 修改未完成的待制作任务。
- 查看待制作列表。
- 将待制作状态更新为已制作。
- 查看鸡尾酒配方和制作说明。

### 3.3 前台访客

- 浏览鸡尾酒菜单。
- 查看鸡尾酒详情。
- 按分类或条件筛选鸡尾酒。

### 3.4 客户用户（一期预留）

- 一期暂不开放客户登录和客户功能。
- 系统设计上需预留用户类型区分能力，便于后续增加客户角色及对应功能。

## 4. 核心业务对象

系统一期建议至少包含以下核心数据对象：

- 鸡尾酒
- 分类
- 标签
- 配方
- 原料
- 图片
- 用户账号
- 用户角色
- 待制作任务

## 5. 功能需求

## 5.1 后台管理端功能（Web）

### 5.1.1 管理员登录

- 支持账号密码登录。
- 支持退出登录。
- 未登录用户不可访问后台页面。
- 登录态应具备基本的会话管理能力。

### 5.1.2 仪表盘

- 显示鸡尾酒总数。
- 显示分类总数。
- 显示原料总数。
- 显示待制作任务数量。
- 显示最近新增或最近更新的鸡尾酒。

### 5.1.3 鸡尾酒管理

- 新增鸡尾酒。
- 编辑鸡尾酒。
- 删除鸡尾酒。
- 查看鸡尾酒列表。
- 支持按名称、分类、状态进行筛选。
- 支持设置是否前台展示。

每条鸡尾酒建议包含以下字段：

- 名称（中文名）
- 英文名
- 简短描述
- 详细介绍
- 基酒类型
- 酒精度说明（可选）
- 杯型
- 口感说明
- 风味标签
- 制作方法
- 装饰说明
- 饮用场景
- 展示状态
- 排序值
- 封面图
- 图集

### 5.1.4 配方管理

- 为每个鸡尾酒维护一组配方明细。
- 支持为一个鸡尾酒添加多条原料记录。
- 每条原料记录需包含原料名称、用量、单位、顺序。
- 支持备注，如“适量”“少许”“点缀”等。

### 5.1.5 分类管理

- 新增分类。
- 编辑分类。
- 删除分类。
- 控制分类是否启用。
- 支持排序。

### 5.1.6 标签管理

- 新增标签。
- 编辑标签。
- 删除标签。
- 为鸡尾酒绑定多个标签。

### 5.1.7 原料管理

- 建立原料基础资料库。
- 支持新增、编辑、删除原料。
- 支持原料分类，如基酒、利口酒、果汁、糖浆、苦精、装饰物。
- 配方录入时可直接选择已有原料。

### 5.1.8 图片管理

- 支持上传鸡尾酒封面图。
- 支持上传多张详情图。
- 支持删除图片。
- 支持替换图片。
- 建议保存图片地址、文件名、上传时间、关联对象。

### 5.1.9 用户与角色管理

- 支持新增后台/前台业务用户。
- 支持编辑用户信息。
- 支持启用和禁用账号。
- 支持设置用户角色。
- 一期角色至少包括管理员、服务员。
- 预留客户角色类型，但一期无需开放对应功能页面。

### 5.1.10 待制作任务管理

- 后台可查看全部待制作任务。
- 支持按状态、创建时间、创建人筛选。
- 可查看任务关联的鸡尾酒、数量、备注、状态。
- 可协助修改任务状态。

### 5.1.11 内容状态管理

- 支持草稿状态。
- 支持已发布状态。
- 支持隐藏状态。

说明：
一期前台公开端默认仅展示“已发布且启用”的内容。

## 5.2 前台公开端功能（移动端优先）

说明：

- 前台公开端为只读菜单入口，用户可直接通过 `/` 或 `/menu` 打开，无需登录。
- 前台公开端与业务前台分离，不承担点单、改状态或登录后任务操作。

### 5.2.1 首页 / 菜单页

- 以菜单形式展示鸡尾酒列表。
- 默认显示鸡尾酒名称、图片、简短描述或风味标签。
- 支持按分类浏览。
- 支持按标签筛选。
- 支持按名称搜索。

### 5.2.2 鸡尾酒详情页

- 展示名称、中英文名。
- 展示封面图和图集。
- 展示详细介绍。
- 展示配方清单。
- 展示制作方法。
- 展示口感、风味、杯型、装饰说明等信息。

### 5.2.3 列表排序

- 支持按后台设置排序值展示。
- 后续可扩展按名称、热度、更新时间排序。

### 5.2.4 响应式展示

- 前台公开端必须优先适配手机屏幕。
- 在常见手机宽度下应保证菜单、详情、图片和按钮可正常阅读与操作。
- 兼容桌面端浏览，但移动端为优先设计目标。

## 5.3 前台业务端功能（管理员 / 服务员登录）

说明：

- 前台业务端统一从 `/app/login` 进入。
- `/app/*` 路由仅允许已登录的 `staff` 或 `admin` 访问，未登录用户不可直接进入。
- 一期中“服务员点单”统一指“创建待制作任务”，不等同于客户在线下单。

### 5.3.1 登录与身份识别

- 支持管理员和服务员通过账号密码登录。
- 登录入口固定为 `/app/login`。
- 登录后根据用户角色展示可访问功能。
- 未登录访问 `/app/*` 时，应拦截并引导回 `/app/login`。
- 一期只开放管理员、服务员相关功能。
- 系统需预留用户类型判断逻辑，后续可以扩展客户角色。

### 5.3.2 待制作任务新增

- 登录用户可以从鸡尾酒列表或详情页发起“加入待制作”。
- 业务上可将该动作理解为“服务员点单”，系统内仍统一记为创建待制作任务。
- 新增待制作任务时，至少包含鸡尾酒、数量、备注信息。
- 可记录下单人或录入人信息。

### 5.3.3 待制作任务编辑

- 登录用户可修改未完成的待制作任务。
- 可修改数量、备注、优先级、状态等字段。
- 已制作完成的任务默认不可随意编辑，避免数据混乱。

### 5.3.4 待制作任务列表

- 登录用户可查看待制作任务列表。
- 支持按状态查看，如待制作、制作中、已制作。
- 支持按时间倒序查看最新任务。
- 后续可扩展按操作人、桌号、客户等条件筛选。

### 5.3.5 待制作任务状态流转

- 一期建议至少包含以下状态：
- 待制作
- 制作中
- 已制作

- 登录用户可以将任务状态从待制作更新为制作中或已制作。
- 任务状态变更应记录操作时间和操作人。

### 5.3.6 配方查看

- 登录后的管理员和服务员可查看鸡尾酒配方。
- 可查看原料、用量、单位、制作方法、装饰说明。
- 详情展示应适合手机端快速查看。

### 5.3.7 前台角色预留设计

- 用户表中需区分角色或用户类型。
- 路由、菜单、接口权限应预留客户角色扩展能力。
- 当前客户角色不展示任何业务功能，但数据库与权限模型需兼容未来扩展。

## 7. 业务规则

- 鸡尾酒必须至少归属一个分类。
- 鸡尾酒可关联多个标签。
- 鸡尾酒应至少包含一条配方记录后才能发布。
- 删除已被配方引用的原料时，应给出提示或限制删除。
- 访客可直接访问 `/` 或 `/menu` 查看公开菜单，无需登录。
- 前台公开端仅显示状态为“已发布”且“可见”的鸡尾酒。
- 前台业务端统一使用 `/app/*` 路由，并要求登录后访问。
- 前台业务端登录用户可查看配方，不受公开展示限制。
- 待制作任务只能由已登录的管理员或服务员创建。
- 已制作任务默认不可直接删除，建议仅允许查看或有限修改。
- 任务状态变更需保留操作记录字段。
- 图片上传后应保存规范的访问地址。

## 8. 非功能需求

### 8.1 性能

- 列表页应支持分页。
- 后台查询应支持基本筛选条件。
- 图片加载应尽量优化，避免影响前台访问速度。
- 移动端页面应尽量减少首屏加载压力。

### 8.2 安全

- 后台接口和前台业务接口都需要鉴权。
- 管理员和服务员密码需加密存储，不可明文保存。
- 不同角色应按权限访问对应功能。
- 图片上传需限制文件类型与大小。

### 8.3 可维护性

- 前后端分离设计优先。
- 数据表结构应便于后续扩展。
- 接口命名与字段命名应保持统一规范。
- 角色权限逻辑应集中管理，避免散落在各页面中。

### 8.4 可扩展性

后续可扩展以下能力：

- 客户登录与客户菜单
- 收藏功能
- 推荐功能
- 原料库存关联
- 酒单专题页
- 多语言内容
- API 对外开放

## 11. 验收标准建议

- 管理员可以登录后台并完成鸡尾酒内容管理。
- 后台可以维护服务员账号。
- 可以录入完整的鸡尾酒信息、配方和图片。
- 前台公开端可以正常展示鸡尾酒菜单和详情，并适配手机端。
- 访问 `/` 或 `/menu` 时，用户可以直接看到公开菜单，无需登录。
- 管理员和服务员可以在前台业务端登录。
- 未登录访问 `/app/tasks` 等业务前台路由时，会被拦截并引导到 `/app/login`。
- 登录用户可以新增、编辑、查看待制作鸡尾酒任务。
- 登录用户可以将任务状态更新为已制作。
- 登录用户可以查看鸡尾酒配方。
- `guest` 始终不可访问 `/app/*`，`admin` 仍是唯一可访问 `/admin/*` 的角色。
- 系统已具备基础角色区分能力，并预留客户角色扩展空间。
- 数据可稳定存储在 MySQL 中。
